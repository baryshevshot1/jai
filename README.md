# jai

Боксовый офлайн ИИ-ассистент для бизнеса. Десктоп-приложение, которое после установки работает **полностью на компьютере пользователя**: без облака, данные не покидают ПК (152-ФЗ), без терминала.

## Стек

- **Tauri 2** (Rust + TypeScript/Vite)
- **Ollama** — единственный движок нейросетей, локально (`127.0.0.1:11434`)
- **sqlite-vec + bge-m3** — поиск по документам (RAG)

## Возможности

**Готово (работает сейчас):**
- Чат с локальной моделью со стримингом ответа; выбор модели; режим «Размышления» (Thinking) по возможностям модели; остановка генерации.
- История диалогов (создание/переключение/удаление/очистка) — хранение на ПК.
- **Документы:** прикрепление файла к сообщению и **база документов с поиском по смыслу (RAG)** — PDF, Word (`.docx`), текст; под ответом видны источники. Всё офлайн.
- **Установка моделей из интерфейса** (например, `bge-m3` для поиска по документам) с прогрессом и отменой — без терминала.
- Подбор моделей под железо («светофор» RAM/CPU/GPU); светлая/тёмная темы; долговечные настройки (тема, модель, Thinking).

**В планах (MVP / Стадия 2):**
- Зрение и распознавание текста/таблиц (OCR) — модель `qwen3-vl`.
- Профильный навык кода.
- Операционный слой целиком: управление процессом Ollama, провижининг базового набора моделей, офлайн-поставка моделей с диска.
- Речь (STT/TTS), агентный режим — Стадия 2.

Актуальный список изменений по фичам — в [`CHANGELOG.md`](./CHANGELOG.md).

## Как поднять проект самому

Разработка и запуск — **только на macOS**.

### Что нужно установить (один раз)

- **Node.js** — версия зафиксирована в [`.nvmrc`](./.nvmrc). С [nvm](https://github.com/nvm-sh/nvm): в папке проекта выполнить `nvm use`.
- **Rust** (компилятор + cargo):
  ```bash
  curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
  ```
- **Ollama** — нужна, только чтобы получать ответы модели. Окно откроется и без неё (в шапке будет «Ollama недоступна»). Запуск движка и хотя бы одна модель:
  ```bash
  ollama serve              # если не запущен как сервис
  ollama pull qwen3.5:9b    # на слабом железе — qwen3.5:4b
  ```

### Запуск

```bash
cd jai
npm install          # только в первый раз (или после смены зависимостей)
npm run tauri dev    # поднимает Vite (порт 1420) и открывает окно приложения
```

Одна команда `npm run tauri dev` делает всё: запускает фронтенд-сервер Vite на
`http://localhost:1420` и компилирует Rust-бэкенд, затем открывает окно. В первый
раз Rust собирается несколько минут, дальше — секунды. Закрыть — закрыть окно или
`Ctrl+C` в терминале.

### Остановка и освобождение памяти

Закрытие окна Tauri **не** выгружает модель: Ollama — отдельный сервис, и модель
ещё ~5 минут висит в RAM/VRAM. Чтобы сразу освободить память и не грузить систему:

```bash
# 1. Закрыть приложение: Ctrl+C в терминале с `npm run tauri dev` (или закрыть окно).
#    Принудительно из другого терминала:
pkill -f 'target/debug/jai'

# 2. Выгрузить модель из памяти, не дожидаясь авто-выгрузки Ollama:
ollama stop qwen3.5:9b      # конкретная модель
# или выгрузить все загруженные сразу:
ollama ps | awk 'NR>1{print $1}' | while read m; do ollama stop "$m"; done

ollama ps                   # проверить, что в памяти ничего не осталось
```

### Проверить только сборку фронтенда

```bash
npm run build        # tsc + vite build, без окна
```

## Сборка установщиков

Под Linux и Windows собираются в **GitHub Actions** (Tauri не умеет кросс-компиляцию) по пушу тега:

```bash
git tag app-vX.Y.Z
git push --tags
```

Готовые установщики появляются в разделе **Releases** (черновик релиза).
