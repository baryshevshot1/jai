Офлайн ИИ-ассистент: процесс разработки MVP

Это руководство не только про то, ЧТО строить, но и про то, КАК идёт разработка: где живёт код, на какой машине что делается, что от чего зависит. Каждый шаг показывает свою связь с соседними. Разработка ведётся на MacBook через Claude Code; готовый продукт под Linux и Windows собирается в облаке и тестируется на чистых машинах.

Разработка — на macOSСборка — GitHub ActionsТест — Linux + WindowsОдин движок: Ollama

◆Что мы строим

Сначала общая картина продукта, потом — модель процесса разработки.

Продукт — десктоп-приложение с иконкой на рабочем столе. Внутри спрятана нейросеть, которая работает полностью на компьютере клиента: без облака, данные никуда не уходят (152-ФЗ). Клиент открыл приложение → оно проверило железо, скачало модели, запустило движок и чат → клиент работает. Никакого терминала у клиента.

Принцип MVP: лёгкий, стабильный, минимум своей разработки

Один движок (Ollama), готовая основа чата, актуальные модели. Цель — не уйти в месяцы разработки, а собрать рабочий продукт и принести клиенту.

Правило, которое экономит недели

«Работает на Mac в разработке» ≠ «работает у пользователя на чистой Windows». Поэтому продукт всегда проверяется как артефакт сборки на чистой машине — контур 2. Разработка (контур 1) только на Mac.

◆Почему нужен GitHub Actions

Не ради удобства — это техническая необходимость.

Tauri официально не умеет кросс-компиляцию: с MacBook нельзя собрать Windows-установщик (.exe/.msi) и надёжный Linux-пакет. Решение индустрии — собирать на чистых облачных машинах через GitHub Actions: матрица из трёх ОС (macos-latest, ubuntu-22.04, windows-latest) официальным tauri-action.

Это и есть главный аргумент за GitHub

Без CI на чистых Windows/Linux-раннерах продукт под эти ОС негде взять. Бонус: Windows-установщик соберётся, даже если у тебя нет Windows-машины.

Про агентный режим (выход в интернет)

Опциональная надстройка над офлайн-ядром: сама модель в сеть не ходит — она через tool-calling просит наш Rust-код выполнить действие (веб-поиск, чтение URL, погода/курсы/новости), а тот возвращает результат. Жёсткие правила: выключен по умолчанию, включается явным тумблером пользователя; в интерфейсе видно, что использован интернет; инструменты контролируемые (а не «модель гуляет по сети»); в 152-ФЗ-чек-листе — отдельный раздел про то, что уходит наружу с согласия пользователя. Офлайн-режим остаётся базовым и полностью локальным. Реализуется ПОСЛЕ офлайн-ядра.
