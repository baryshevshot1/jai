# Документация jai

## Проектные документы (фаза проектирования)
- **MVP_руководство_процесс_Linux_Windows.html** — пошаговое руководство по разработке (11 фаз, 42 шага).
- **Инженерный_отчёт_финальный.html** — инженерный отчёт: ключевые решения и набор моделей.
- **jai_дизайн_макет.html** — **базовый** HTML-прототип интерфейса.

Это снимки фазы проектирования (открываются двойным кликом в браузере) — обоснование
решений и план, **а не финальный вид продукта**.

> ⚠️ **Важно:** `jai_дизайн_макет.html` — это исходный **базовый прототип**. После него
> интерфейс и логика получили значительный объём правок и доработок (редизайн в несколько
> этапов, карточки прикреплённых файлов, вкладки «Диалоги | Документы», RAG-источники под
> ответами, установка моделей из интерфейса, доработки UX и т. д.). Поэтому прототип и
> реальное приложение **отличаются** — ориентируйся на код и на [`../CHANGELOG.md`](../CHANGELOG.md),
> а макет используй как отправную точку замысла, а не как актуальный эталон.

**Актуальное состояние реализации** см. в журнале изменений
[`../CHANGELOG.md`](../CHANGELOG.md).

## Отчёты по этапам
