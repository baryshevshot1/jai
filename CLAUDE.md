## Что это за проект

**jai** — боксовый офлайн ИИ-ассистент для российского бизнеса. Десктоп-приложение: клиент открывает по иконке, оно работает **полностью на его компьютере** (без облака, данные не покидают ПК — 152-ФЗ), без терминала. Бизнес-модель — установка под ключ.

## Стек (фактический)

- **Оболочка:** Tauri 2 (Rust) — окно, упаковка под Linux/Windows.
- **Фронтенд:** TypeScript + **Vite** (шаблон `vanilla-ts`). Dev-сервер на `http://localhost:1420`.
- **Бэкенд:** Rust (`src-tauri/`) — детект железа, менеджер моделей, контроль движка.
- **Движок нейросетей:** ТОЛЬКО **Ollama**, локальный HTTP API на `http://127.0.0.1:11434`.
- **RAG/эмбеддинги:** sqlite-vec (один файл БД) + модель bge-m3.
- Идентификатор приложения: `ru.baryshevshot.jai`.

## Жёсткие правила (не нарушать без явного запроса)

1. **Единственный движок — Ollama.** НЕ добавлять второй движок (llama.cpp и т.п.) без явного запроса. Речь (STT/TTS) — это Стадия 2, не сейчас.
2. **Сеть — только localhost.** Никаких исходящих соединений с пользовательскими данными. Ollama слушает только `127.0.0.1`, никогда `0.0.0.0`.
3. **Персональные данные не покидают ПК.** Всё хранение и вычисления — локально.
4. **Офлайн после установки.** Приложение должно работать без интернета (модели уже скачаны). Проверки сети — неблокирующие.
5. **Никаких секретов в коде/гите.** Ключи подписи, токены, `.env` — мимо репозитория (см. `.gitignore`).

## Архитектура (целевая)

```
Tauri-окно (TypeScript + Vite: чат, выбор модели, светофор железа, проверка системы)
        ↕ invoke / emit-listen
Rust-бэкенд (src-tauri):
  • Детектор железа     — RAM, CPU, видеокарта (Windows: DXGI), VRAM
  • Менеджер моделей     — качает из реестра Ollama и с Hugging Face, проверяет, ставит
  • Контроль движка      — наличие/версия Ollama, порт, гасит СВОЮ группу процессов
        ↕ только http://127.0.0.1:11434
Ollama (один движок): текст, русский, код, зрение+OCR (qwen3-vl), эмбеддинги (bge-m3)
        ↕
Данные на ПК пользователя: история, индекс sqlite-vec, модели — ничего не уходит наружу
```

## Набор моделей (зафиксирован)

| Навык | Модель · тег | Источник |
|---|---|---|
| Текст (база) | `qwen3.5:9b` (fallback `:4b`) | реестр Ollama |
| Русский (профиль, опц.) | `hf.co/t-tech/T-lite-it-2.1-GGUF:Q4_K_M` | Hugging Face |
| Зрение + OCR | `qwen3-vl:8b` (или `:4b`) | реестр Ollama |
| Документы (RAG) | `bge-m3` + sqlite-vec | реестр Ollama |
| Код | `qwen3-coder` | реестр Ollama |

- **Важно:** русские модели T-tech **НЕ в реестре Ollama** под `t-tech/` — они на Hugging Face. Ставить так: `ollama run hf.co/t-tech/T-lite-it-2.1-GGUF:Q4_K_M`. Команда `ollama pull t-tech/...` не работает.
- Базовый набор: `qwen3.5:9b` (+`:4b` на слабом железе) + `bge-m3`. Остальное — под роль.
- Минимальная версия Ollama: эффективный минимум = наибольшее из требований (зрение qwen3-vl ≥ 0.12.7; безопасность ≥ 0.17.1); ориентир — ветка 0.30.x.

## Целевое железо (тест)

MacBook (разработка) · Linux-ПК с NVIDIA 6 ГБ VRAM / 64 ГБ RAM (тест собранного продукта). Модель 8–9B (Q4_K_M) на 6 ГБ — впритык, держать контекст ≤ 8k.

## Кроссплатформенность (важно при написании кода)

Продукт собирается под **Linux и Windows** (и macOS на будущее), хотя разрабатывается на Mac. Поэтому весь код должен быть портируемым:

- **Пути — только через Tauri path API** (`appDataDir`, `appConfigDir`, `appLogDir` и т.п.). Никогда не хардкодить разделители (`/` или `\`) и абсолютные пути.
- **ОС-специфичный код Rust — через `#[cfg(target_os = "...")]`** (`windows` / `linux` / `macos`). Не предполагать одну ОС.
- **Детект видеокарты/VRAM — отдельные ветки:** Windows — DXGI (крейт `windows`); Linux — `nvidia-smi` / `rocm-smi`; macOS — Metal/unified memory. Не использовать WMI `AdapterRAM` (режет до 4 ГБ).
- **Управление процессом движка** (запуск/остановка Ollama, освобождение порта) — учитывать различия Windows и Unix; гасить свою группу процессов кроссплатформенно.
- **Windows-установщик — офлайн:** `webviewInstallMode: offlineInstaller` уже задан в `tauri.conf.json` (WebView2 ставится без интернета). На Linux это не требуется (системный webkit2gtk).
- **Сборка установщиков — только в CI** (`.github/workflows/release.yml`, matrix из 3 ОС). Tauri не кросс-компилит — с Mac собрать Windows/Linux напрямую нельзя.
- Не завязываться на команды/утилиты, специфичные для одной ОС, без `#[cfg]`-ветки и проверки наличия.
